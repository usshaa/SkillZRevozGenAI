1

User Input

The user authenticates via the Streamlit frontend. Upon logging in, the system determines their Role-Based Access Control (RBAC) privileges (e.g., Customer vs. Admin). The user then submits a raw natural language query into the chat interface.

Technical Specifications:

  • Framework: streamlit >= 1.30.0
  • UI Components: st.chat_input, st.chat_message
  • Auth Mapping: Evaluates st.session_state.username against a predefined roles dictionary.
2

State Hydration (Context Injection)

Before sending the query to the AI, Streamlit packages the raw text alongside the user's secure metadata (Name, Role, Customer ID). This data is dynamically injected as an immutable `customer_context` variable into the core LangGraph `AgentState` on every request. This structurally overrides LLM hallucinations regarding identity.

Technical Specifications:

  • Schema: AgentState(TypedDict)
  • Context Var: customer_context: str
  • Deduplication: Uses Annotated[Sequence[BaseMessage], add_messages] to merge message IDs.
3

Semantic Routing

The populated `AgentState` is passed to the LangGraph Supervisor. The Supervisor is a specialized LLM node tasked exclusively with intent recognition. It reads the latest message, ignores attempting to answer it, and outputs a singular routing token to trigger a conditional edge traversal (`faq`, `order`, or `refund`).

Technical Specifications:

  • Model: llama-3.1-8b-instant via ChatGroq API
  • Node Logic: supervisor_node(state) returning {"next": token}
  • Routing: workflow.add_conditional_edges evaluates the returned token.
4

Execution (Tools & RAG)

The specialized worker agent receives the query and executes its objective. The FAQ Agent queries Qdrant using Hybrid Search (Dense Vectors + BM25 Sparse Index). Alternatively, the Order/Refund Agents halt text generation, format JSON tool calls, and utilize Python Langchain ToolNodes to execute exact-match, parameterized SQL queries against the local SQLite database.

Technical Specifications:

  • Dense Vectors: SentenceTransformers('all-MiniLM-L6-v2') (384-dims)
  • Sparse Vectors: BM25Okapi via rank_bm25
  • Tooling: @tool bound Python functions executing parameterized sqlite3.connect queries.
5

Synthesis

Once the raw data is retrieved from either Qdrant or SQLite, it is piped directly back into the worker agent's context window. The agent synthesizes this raw JSON or text block into a polished, natural language conversational response tailored to the user's specific context.

Technical Specifications:

  • Payloads: Raw tool data is appended as a ToolMessage class.
  • ReAct Loop: Agent LLM parses the ToolMessage array to synthesize a final AIMessage string.
6

Absolute State Persistence

Upon completion of the graph traversal, the `langgraph-checkpoint-sqlite` engine intercepts the `AgentState`. It serializes the absolute entirety of the chat history, tool executions, and identity parameters into a binary BLOB. This is safely committed to the `novashop_memory.db` file keyed to the user's `thread_id`.

Technical Specifications:

  • Library: langgraph-checkpoint-sqlite >= 3.1.0
  • Instantiation: SqliteSaver(conn) with explicit .setup() call.
  • Concurrency: Uses -wal (Write-Ahead Logging) and -shm for concurrent SQLite operations.
7

Output to User

The state engine yields the final synthesized string back to the Streamlit frontend. The UI extracts the AI's content block, updates the visual chat array, and renders the instantaneous response for the user to read.

Technical Specifications:

  • Streaming: Graph checkpointer yields the final state mapping.
  • Rendering: Streamlit executes st.markdown() to paint the response into the chat container.
